## Key Components

### Frontend (index-tw.blade.php)
- **Button**: Conditionally rendered when filters are active (`@if($hasFilter)`)
- **JavaScript**: Reads filter state from Blade variables, builds URL, copies to clipboard
- **Visual Feedback**: Button changes to "âœ“ Copied!" for 2 seconds

### Backend (EventsController.php)
- **applyFilterFromUrl()**: New method that:
  1. Initializes session store
  2. Extracts filters from URL query parameters
  3. Saves filters to session
  4. Redirects to filter route

### Routes (web.php)
- **GET /events/apply-filter**: Maps to `applyFilterFromUrl()` method

### Session Flow
```
URL Params â†’ Controller â†’ Session Store â†’ Redirect â†’ Filter Method â†’ Session â†’ Query Builder â†’ Results
```

## Filter Parameter Examples

### Simple Text Filter
```
?filters[name]=Concert
```

### Array Filter (Tags)
```
?filters[tag][]=music&filters[tag][]=live
```

### Nested Object (Date Range)
```
?filters[start_at][start]=2024-01-01&filters[start_at][end]=2024-01-31
```

### Combined with Sorting
```
?filters[name]=Concert&sort=start_at&direction=desc&limit=25
```
